## 印象に残った発表

### LT:なんでRustの環境構築してないのにRust製のツールが動くの？

スピーカー： 株式会社ZOZO フロントエンドエンジニア ssssotaさん

#### 概要

現在のフロントエンド開発現場で使われるリンターやフォーマッターなどのツールチェーンはRust製の普及していますが、JavaScript, TypeScriptを使っているフロントエンドエンジニアは特にRustの環境構築を行なっているわけではありません。その中でRustのバイナリがどのようにJavaScriptエコシステムで動いているのかを解説しているLTでした。

#### 感想・学び

Rust製ツールが動く仕組みは主に、

- CLIベース
- Node APIベース
- Wasm(WebAssembly)ベース

の3つのパターンがあるようです。

DenoやBiomeなどはCLIベースで動いていて、ユーザーはOSとCPUアーキテクチャにあったバイナリをインストールし、CLIとして直接実行する仕組みのようです。この仕組みは自分自身もよく見るような動作だなと感じていました。
Node APIベースでは、Rustで書かれているネイティブのバイナリが、Node.jsと通信しながら動作する仕組みでした。
Node.jsでは、C++やRustなどで書かれたネイティブアドオンと連携するためのNode APIという仕組みが用意されているようです。
Node.jsは使うことが多いですが、Node APIは全く知らなかったので勉強になりました。
Wasmベースは、Rustのコードをコンパイルしてブラウザ上で動作させる仕組みです。WasmはOSやCPUに依存しないのでブラウザ上で動作可能というメリットがありますが、ネイティブバイナリに比べて若干実行速度が劣るようです。
自分が最近使っているPythonでもuvやRuffなどRust製ツールは多いなと感じていたので、これからソフトウェア開発全体でRust製ツールがどんどん標準的になっていくのかなと感じました。

### LT:React Nativeならぬ"Vue Native"が実現するかも？新世代マルチプラットフォーム開発フレームワークのLynxとLynxのVue.js対応を追ってみよう

スピーカー：シニアコンサルタント　フューチャーアーキテクト株式会社 永井優斗さん

#### 概要

TikTokを運営するByteDance社が開発しているモバイル向けマルチプラットフォーム開発フレームワークであるLynxの紹介とVue.js対応についてのコミュニティの動向の紹介LTでした。

#### 感想・学び

現在は、Reactのみを公式サポートしているようですが、Vue.jsの作者であるEvan You氏がVue対応を支援するXで意向を表明しています。コミュニティでは、Vue+LynxのプロトタイプをGitHubで公開しているようですが、最近はリポジトリの更新が少なく、開発が難航しているようです。
もし、Vue+Lynxが実現すると、Vue構文でネイティブアプリ開発ができるようになり、ReactでいうReact NativeのようなVueネイティブアプリ開発の決定的な選択肢になりうる可能性があると解説していました。
Vue自体フロントエンドフレームワークの中では学習コストが少ないものとよく紹介されているので、LynxのVue対応が実現すれば更にモバイルアプリ開発がやりやすくなる未来があると感じました。
また、最近Xなどインターネット上でヒューマノイドやLLMなど中国の高い技術力が目立つように見えます。Web技術においても中国のOSSが広がっていく未来もあるのかなと感じました。

### LT:chocoZAPサービス予約システムをNuxtで内製化した話

スピーカー：フロントエンドエンジニア RIZAPテクノロジーズ株式会社 Kaede Katoさん

#### 概要

24時間無人ジムであるチョコザップのセルフエステやネイルサービスのアプリ予約システムを外部のものから、Nuxtを用いて内製化したというLTでした。その中でも、技術構成と認証周り、モバイルアプリ内のWebビューの工夫についての解説が多くありました。

#### 感想・学び

技術構成は、

- フロントエンド：Nuxt.js
- BFF(Backend For Frontend)：Nuxtサーバー
- バックエンド：Rails API

という構成となっていました。

Nuxtサーバーがセッション検証などを行なった上でバックエンドAPIを呼び出す構成となっているようです。これによって、レスポンスをフロントエンド向けに整形でき、バックエンドのエンドポイントも外部に公開せずに済むためセキュリティが向上するという解説がありました。
この構成は初めて知ったのですが、バックエンドをRuby on Railsで統一しているRIZAPテクノロジーズに適したシステム構成なのかなと感じました。
また、Webビュー特有の問題としてiOSの一部端末で、アプリ側の戻るボタンを押すと、最初のページに戻ってしまう不具合があったようです。これはアプリ側がNuxtのページ遷移履歴を認識できていないことが原因だったと解説されていました。自分は全くモバイルアプリ開発をしたことがないのですが、Webビューでモバイルアプリを開発するときにこのような問題が発生すると分かって、Web技術でモバイルアプリを開発するのも一長一短なんだなと感じました。

## 今後の展望

今回のVueFesJapanの参加を通して、VueやNuxtを使ったWeb開発に更にチャレンジしていきたいと感じました。また、JavaScriptエコシステムを支えているツールについても様々知ることができたため、それらの使い方だけではなく仕組みも勉強しながら開発を進めていきたいと感じました。今後は、内定者研修でVueとNuxtについてしっかり学びながら、個人開発でも積極的に取り入れていこうと考えています。
